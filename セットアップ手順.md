# 西町GP前店 在庫管理システム セットアップ手順

---

## システム概要

```
仕入れ都度入力
  └─ Googleフォーム（仕入れ記録）
       └─ GASトリガー → 仕入れログシートに自動記録
                              ↓
月1回の棚卸
  └─ 棚卸シート（月次）
       ├─ 繰越在庫：前月末在庫を自動引継ぎ
       ├─ 当月仕入れ：仕入れログから自動集計（SUMIF）
       ├─ 使用数：月1回手入力（黄色セル）
       └─ 現在庫・在庫金額：自動計算
```

---

## STEP 1：Googleスプレッドシートにスクリプトを設定

1. Googleスプレッドシートを新規作成
2. メニュー「拡張機能」→「Apps Script」を開く
3. `Code.gs` の内容を貼り付けて保存（Ctrl+S）
4. スプレッドシートに戻り、メニューが更新されるのを待つ
5. 上部メニュー「在庫管理」→「🆕 初期セットアップ」を実行

→ `商品マスタ` と `仕入れログ` シートが自動作成されます

---

## STEP 2：商品マスタに商品情報を入力

`商品マスタ` シートに全商品を入力します。

| 列 | 項目 | 例 |
|---|---|---|
| A | 発注先 | ファンセクト |
| B | カテゴリ | フード |
| C | 商品種別 | わらびもち |
| D | 商品名 | わらびもち（必須・ユニーク） |
| E | 規格・サイズ | 5kg*4枚 |
| F | 入数/1LOT | 1箱 |
| G | 単位 | 箱 |
| H | 単価(税抜) | 12550 |
| I | 消費税区分 | 軽8% または 標準10% |

> ⚠️ **商品名（D列）は他の商品と重複しないようにしてください。**  
> 仕入れログとの照合キーになります。

---

## STEP 3：棚卸入力アプリをデプロイ（ウェブアプリ公開）

1. Apps Script エディタを開く
2. 右上「デプロイ」→「新しいデプロイ」をクリック
3. 歯車アイコン→「ウェブアプリ」を選択
4. 以下のように設定：

| 設定項目 | 値 |
|---|---|
| 説明 | 棚卸入力アプリ |
| 次のユーザーとして実行 | 自分 |
| アクセスできるユーザー | 自分のGoogleアカウント（社内のみ）または「全員」 |

5. 「デプロイ」をクリック → 表示されたURLをコピー
6. スプレッドシートのメニュー「在庫管理」→「📦 棚卸入力アプリを開く」でもURLを確認できます
7. URLをスタッフのスマホでブックマーク登録してもらう

> ⚠️ Code.gs を修正したあとは「デプロイ」→「デプロイを管理」→「編集（鉛筆アイコン）」→バージョン「新しいバージョン」→「デプロイ」で更新してください。

---

## STEP 3-2：GitHub Pages で棚卸画面（index.html）を公開する

> この方式では、**APIはGAS（Code.gs）**、**画面はGitHub Pages（index.html）** で動きます。

1. このフォルダをGitHubリポジトリにpush（`index.html` を含む）
2. GitHubの「Settings」→「Pages」→
   - Source: `Deploy from a branch`
   - Branch: `main`（または利用中のブランチ）/ `/ (root)`
3. 公開URL（例: `https://<user>.github.io/<repo>/`）を確認
4. STEP 3で取得したGASのWebアプリURL（`.../exec`）を使って、次の形式でアクセス

   `https://<user>.github.io/<repo>/?api=<GAS_WEB_APP_URL>`

   例:
   `https://example.github.io/inventory-app/?api=https://script.google.com/macros/s/xxxxxxxxxxxxxxxx/exec`

5. スタッフにはこのGitHub Pages URL（`?api=...` 付き）を共有

> ⚠️ GitHub Pagesは静的ホスティングのため、API自体は動かせません。  
> APIは引き続きApps Script（ウェブアプリ）としてデプロイしてください。

---

## STEP 4：仕入れフォームを自動作成

1. スプレッドシートのメニュー「在庫管理」→「📋 仕入れフォームを作成」を実行
2. 初回はGoogleアカウントへの権限許可が求められます→「許可」をクリック
3. 完了ダイアログに **入力用URL** と **編集用URL** が表示されます
   - **入力用URL** → スタッフにLINE/メールで共有する
   - **編集用URL** → 管理者がフォームを編集する場合に使用

フォームには以下の質問が自動で作成されます：

| 質問名 | 形式 | 備考 |
|---|---|---|
| 納品日 | 日付 | 必須 |
| 伝票No. | 短文 | 任意 |
| 商品名 | プルダウン | 商品マスタの全商品が自動セット |
| 数量(LOT) | 数値 | 必須 |
| 備考 | 段落 | 任意 |

> ✅ トリガー（フォーム送信→自動記録）も同時に設定されます。手動設定不要です。

---

## STEP 5：フォームURLをスタッフに共有

1. STEP 3 で表示された **入力用URL** をコピー
2. LINEグループやメールでスタッフに送る
3. スマートフォンのホーム画面にショートカット追加を案内すると便利です

> 📌 商品が増えた場合は、商品マスタに追加後「🔁 フォームの商品リストを同期」を実行するとプルダウンが自動更新されます。

---

## STEP 5：初月の棚卸シートを作成

1. スプレッドシートのメニュー「在庫管理」→「📦 翌月棚卸シートを作成」を実行
2. `棚卸_YYYYMM` という名前のシートが自動作成されます

> 初月のみ繰越在庫が0でスタートします。  
> 実際の繰越在庫がある場合は I列（繰越在庫）を手入力してください。

---

## 月次の運用フロー

### 仕入れ時（都度）
1. Googleフォームを開く（スマホからもOK）
2. 納品日・伝票No.・商品名・数量を入力して送信
3. 自動的に `仕入れログ` シートに記録される
4. 棚卸シートの「当月仕入れ」列に即時反映される

### 月末棚卸（月1回）
1. スマホで**棚卸入力アプリ**のURLを開く
2. 商品ごとに実際に数えた在庫数を入力（繰越・仕入れ数が参考表示される）
3. 全商品入力したら「一括送信」ボタンを押す
4. スプレッドシートの棚卸シートに自動反映、使用数も自動計算される
5. メニュー「在庫管理」→「📅 翌月の棚卸シートを作成」を実行
6. 翌月シートに今月末在庫が自動引き継がれる

## よくある質問

**Q: 商品を追加したい**  
A: 商品マスタにを追加すれば、翌月から棚卸シートに自動で反映されます。

**Q: 単価が変わった**  
A: 商品マスタのH列（単価）を更新してください。翌月の棚卸シートから新単価が反映されます。

**Q: 今月の仕入れが反映されない**  
A: メニュー「在庫管理」→「🔄 今月の仕入れ合計を再集計」を実行してください。
